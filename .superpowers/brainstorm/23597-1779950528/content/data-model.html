Data Model Changes

New fields, new tables, and how they connect

Existing table — subjects
subjects
id uuid PK
user_id uuid FK
name text
color text
created_at timestamptz
+ exam_board text nullable — e.g. "Edexcel IAL", "OCR A"
+ target_grade text nullable — e.g. "A*", "90%", "7"
New tables — subject labels (grouping tags)
subject_labels
id uuid PK
user_id uuid FK
name text
⟷
subject_label_map
subject_id uuid FK
label_id uuid FK
PK: (subject_id, label_id)

Labels are reusable across subjects. "A-Level" defined once, applied to Physics, Maths, Chemistry, etc. This is a separate concept from session tags.

New route
/ timer · / stats · / notes · new / subjects / :id
— hub page for a single subject
Example — Physics subject object (after migration)
